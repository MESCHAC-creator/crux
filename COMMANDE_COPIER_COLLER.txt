## 📋 ÉTAPES INDIVIDUELLES (si tu préfères)

# 1. Aller au répertoire

# 2. Initialiser git
git init

# 3. Configurer ton identité
git config user.name "Ton Nom"
git config user.email "ton@email.com"

# 4. Ajouter GitHub (remplace TON_USERNAME)
git remote add origin https://github.com/TON_USERNAME/crux.git

# 5. Vérifier le remote
git remote -v

# 6. Ajouter les fichiers
git add .

# 7. Vérifier (optionnel)
git status

# 8. Commiter
git commit -m "🚀 Release CRUX v1.0.0 - Production Ready"

# 9. Créer un tag
git tag -a v1.0.0 -m "CRUX v1.0.0 Production Release"

# 10. PUSH VERS GITHUB
git push -u origin main

# 11. PUSH LE TAG
git push origin v1.0.0

# 12. Vérifier le succès
git log --oneline -3
git tag -l

## 📊 CE QUI VA SE PASSER:

Quand tu exécutes la commande:

1. cd /chemin/vers/crux
   → Vérifier que tu es au bon endroit

2. git init
   → Créer le dossier .git

3. git config user.name/email
   → Configurer qui tu es

4. git remote add origin ...
   → Dire à git où envoyer les fichiers

5. git add .
   → Sélectionner tous les fichiers

6. git commit -m "..."
   → Créer un "snapshot" avec message

7. git tag -a v1.0.0 -m "..."
   → Marquer cette version comme v1.0.0

8. git push -u origin main
   → Envoyer les fichiers sur GitHub

9. git push origin v1.0.0
   → Envoyer le tag aussi

10. git log --oneline -3
    → Montrer que tout a marché
