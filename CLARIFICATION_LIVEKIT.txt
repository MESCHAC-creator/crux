❓ Ta question: "c'est bien livekit qui est utilisé pour les videos non?"

✅ RÉPONSE: NON

CRUX utilise actuellement:
├─ **AGORA** (pour ≤300 participants)
└─ **JITSI** (pour 300-1000+ participants)

────────────────────────────────────────────────────────────────────────────

POURQUOI CETTE ARCHITECTURE?

Agora (≤300):
  ✅ Latence ultra-basse (50-100ms)
  ✅ Qualité vidéo excellente
  ✅ Infrastructure gérée (zéro ops)
  ✅ Support enterprise Agora
  ❌ Plus cher pour gros volumes

Jitsi (300-1000+):
  ✅ Open-source (contrôle total)
  ✅ Scalable infiniment
  ✅ Auto-hébergeable
  ✅ Gratuit si self-hosted
  ❌ Latence plus haute (200-500ms)

────────────────────────────────────────────────────────────────────────────

COMPARAISON: AGORA vs JITSI vs LIVEKIT

┌─────────────┬──────────┬─────────┬──────────┐
│ Aspect      │ Agora    │ Jitsi   │ LiveKit  │
├─────────────┼──────────┼─────────┼──────────┤
│ Max users   │ 300      │ 1000+   │ 10,000+  │
│ Latency     │ 50ms ⭐  │ 200ms   │ 100ms ⭐ │
│ Cost        │ $$$$     │ Free    │ Free*    │
│ Open Source │ ❌       │ ✅      │ ✅       │
│ Self-host   │ ❌       │ ✅      │ ✅       │
│ Ops burden  │ Low      │ High    │ High     │
└─────────────┴──────────┴─────────┴──────────┘

* LiveKit is free but requires you to manage servers

────────────────────────────────────────────────────────────────────────────

POURQUOI PAS LIVEKIT D'EMBLÉE?

LiveKit c'est excellent, MAIS:

1. Nécessite infrastructure self-hosted
   • Serveur Linux
   • TURN server
   • Redis
   • Load balancer
   = ~$200/mois minimum

2. Nécessite expertise DevOps
   • Déploiement
   • Monitoring
   • Scaling
   • Maintenance

3. Moins testé à grande échelle que Agora

4. Pas de support commercial inclus
